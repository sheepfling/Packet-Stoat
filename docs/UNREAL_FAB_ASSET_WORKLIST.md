# Unreal Fab Asset Worklist

This worklist tracks the authored Unreal assets and captured evidence that keep
Epic 1 at `fab_ready`.

The source-backed runtime shell is verified by `FastDis.Demo.FabSourceShell`.
This page only tracks assets that must be authored or captured in Unreal. It is
now a maintenance checklist: if any required asset is removed or invalidated,
`tools/check_unreal_fab_readiness.py --strict` should fail.

Use the repeatable generator before hand-authoring polish:

```bash
python tools/create_unreal_fab_demo_assets.py --engine-version 5.8
```

Dry-run the editor command without writing assets:

```bash
python tools/create_unreal_fab_demo_assets.py --engine-version 5.8 --dry-run
```

## Required Verification

Before claiming or preserving `fab_ready`:

- `python tools/check_unreal_fab_readiness.py --strict` passes.
- `python tools/unreal_workflow.py build --engine-version 5.8` passes.
- `python tools/unreal_workflow.py demo --engine-version 5.8` passes.
- The packaged plugin opens in a clean project without relying on repository
  build scripts.
- The demo map can be opened and exercised from the packaged plugin content.
- Screenshots are real captures from the demo map, not synthetic SVG exports.

## Claim Rules

- Do not claim Fab readiness if this page has missing required assets or
  `tools/check_unreal_fab_readiness.py --strict` fails.
- Do not claim full DIS body decoding from event markers; they prove surfaced
  Blueprint summaries and raw-sidecar preservation.
- Do not commit derived marketplace PNG/JPEG exports unless intentionally
  promoted; source visuals stay under `media/storefront/fab/`, and generated
  exports normally stay under `build/`.
