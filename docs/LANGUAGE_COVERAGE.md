# Language Coverage

fastdis has two levels of DIS message support:

- **Catalog support:** the PDU type is known by metadata generated from
  Open-DIS `DIS6.xml` and `DIS7.xml`.
- **Body decoder support:** fastdis parses the PDU body into typed C/C++/Python
  records.

Alpha 2 exposes the generated catalog across C, C++, and Python. Unreal and
Godot can consume the same catalog headers because their adapters use the native
C/C++ headers.

## Current Matrix

| Surface | Catalog support | Body decoder support |
|---|---|---|
| C | `fastdis/fastdis_pdu_catalog.h` | Entity State fixed prefix / transform only |
| C++ | `fastdis/fastdis_pdu_catalog.hpp` | Entity State wrappers over the C ABI only |
| Python | `fastdis.PDU_CATALOG`, `find_pdu`, `known_pdu_types` | Entity State ctypes/native value wrappers only |
| Unreal | Catalog available through staged headers | Entity State transform/snapshot application only |
| Godot | Catalog available through staged headers | Entity State transform/snapshot application only |
| Unity | No adapter yet | No adapter yet |

Known PDU metadata does not mean the PDU body can be parsed. Unsupported body
decoders remain explicitly marked with `has_body_decoder == false`.

## Generated Sources

The catalog is generated by:

```bash
python tools/generate_pdu_catalog.py
```

Inputs:

- `references/open-dis/DIS6.xml`
- `references/open-dis/DIS7.xml`

Outputs:

- `include/fastdis/fastdis_pdu_catalog.h`
- `include/fastdis/fastdis_pdu_catalog.hpp`
- `src/fastdis/pdu_catalog.py`
- `docs/DIS_PDU_CATALOG.md`

## Next Body Decoders

The recommended next body decoders remain:

- Fire PDU
- Detonation PDU
- Collision PDU
- Entity State Update PDU
- Electromagnetic Emission prefix

These should be added incrementally with fixed-prefix tests and no C++ classes
crossing the shared-library ABI.
